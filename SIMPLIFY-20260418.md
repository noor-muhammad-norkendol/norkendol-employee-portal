## What Changed

### 1. Consolidated 4 identical useSupabase hooks into 1 shared hook
**New file:** `src/hooks/useSupabase.ts`
**Modified:** `src/hooks/onboarder-kpi/useSupabase.ts`, `src/hooks/estimator-kpi/useSupabase.ts`, `src/hooks/claim-health/useSupabase.ts`, `src/hooks/settlement-tracker/useSupabase.ts`
**What:** Created one `useSupabase()` hook at `src/hooks/useSupabase.ts`. The 4 old hooks (`useOKSupabase`, `useEKSupabase`, `useCHSupabase`, `useSTSupabase`) now re-export the shared hook as aliases — so nothing that imports the old names breaks.
**Why:** All 4 hooks were identical copy-paste (create Supabase client, fetch user profile). One source of truth means one place to fix bugs or add features.
**If it breaks:** Any page using Supabase auth/user info might fail. Revert the 4 hook files and delete `src/hooks/useSupabase.ts`.

### 2. Created shared formatters — formatDate, formatCurrency, timeAgo, truncate
**New file:** `src/lib/formatters.ts`
**Modified:** 13 files (formatDate), 6 files (formatCurrency), 3 files (timeAgo), 3 files (truncate)
**What:** Consolidated `formatDate` (was in 13 files), `formatCurrency` (6 files), `formatCurrencyDecimal` (3 files), `timeAgo` (3 files), and `truncate` (3 files) into one shared file. Removed local definitions and added imports.
**Why:** Same function copy-pasted everywhere. If you need to change date format, you'd have to find and update 13 files.
**If it breaks:** Dates, currency, or "time ago" labels might look different or show "—". Revert by restoring local function definitions from git history.

**If it breaks:** Status badges on Onboarder KPI or Estimator KPI pages might lose their color. Check that colored pill badges still show correctly. Revert `.text` back to `.color` in the 5 files listed.

### 6. checkContractorInTPN — server-side phone filtering
**Modified:** `src/app/dashboard/onboarder-kpi/page.tsx`
**What:** Changed the TPN contractor phone lookup from fetching ALL external contacts into memory to using `.not("phone", "is", null).ilike("phone", "%<last7digits>%")` to pre-filter server-side.
**Why:** Was loading every external contact in the org just to find one phone number. With 83 users now it's fine, but this doesn't scale. The server-side filter reduces data transferred.
**If it breaks:** When adding a new onboarding client with a contractor phone number, the "already in TPN" detection might miss matches. Revert by removing the `.not()` and `.ilike()` filters and changing `phoneContacts` back to `allContacts`.

### 7. Duplicate file number generation — deduplicated

**Why:** Same DB query and formatting logic was written twice. Bug fixes to one wouldn't fix the other.
**If it breaks:** File numbers might not auto-generate when you type a state code in the Add Client form. Revert the useEffect back to its inline implementation from git history.

### 8. Removed leftover console.log statements
**Modified:** Multiple files (primarily `KPIAdminTab.tsx`)
**What:** Removed debug `console.log` calls left from development/debugging sessions.
**Why:** Clutters browser console in production. No functional impact.
**If it breaks:** It won't — these were debug output only.

### 9. Fixed createClient() per-render issue in 20 files + 3 stale-auth bugs
**Modified:** 20 files across components/ and app/dashboard/
**What:** Every component was calling `createClient()` on every React re-render, creating a new Supabase client each time. Changed to `const [supabase] = useState(() => createClient())` so the client is created once when the component first loads. Also fixed 3 files (ai-agents, my-settings, system-settings) that had `createClient()` at module scope — these held stale auth if a user logged out and back in.
**Why:** Unnecessary work on every render. The module-scope ones were an actual bug — they wouldn't pick up a new login session.
**If it breaks:** Any page might fail to connect to Supabase. Revert `const [supabase] = useState(() => createClient())` back to `const supabase = createClient()` in the affected file. For the 3 module-scope ones, move `const supabase = createClient()` back above the component function.

### 10. XLSX dynamic import — reduced bundle by ~7 MB
**Modified:** `src/app/dashboard/tpn-admin/page.tsx`
**What:** Changed `import * as XLSX from "xlsx"` (static, loaded on every page visit) to `const XLSX = await import("xlsx")` inside the file upload handler (loaded only when admin clicks Import).
**Why:** The xlsx library is 7.2 MB. Every user visiting the TPN Admin page was downloading it even if they never imported a spreadsheet.
**If it breaks:** The Import from XLSX button might fail. Revert `const XLSX = await import("xlsx")` back to a static `import * as XLSX from "xlsx"` at the top of the file, and change the `reader.onload = async (evt)` back to `reader.onload = (evt)`.

### 11. KPIAdminTab — user-visible save error feedback
**Modified:** `src/app/dashboard/executive-intelligence/KPIAdminTab.tsx`
**What:** Added `saveError` state. The catch block now shows "Save failed. Please try again." in red next to the Save button instead of silently failing.
**Why:** Previously the save button would just stop spinning with no indication that it failed. The user would think it worked.
**If it breaks:** Revert by removing the `saveError` state, the `setSaveError` calls, and the `{saveError && ...}` JSX.

### 12. Middleware — don't lock out active users on DB errors
**Modified:** `src/middleware.ts`
**What:** Now checks for `profileError` separately from "user not active." If the DB query fails (network blip, Supabase outage), the user is let through to the dashboard instead of being redirected to `/pending`.
**Why:** Before this fix, a Supabase hiccup would lock every active user out of the portal with no error message — they'd just see the pending page.
**If it breaks:** If you want the strict behavior back (deny access when DB is unreachable), change `if (profileError || !profile) { return response; }` back to `if (!profile || profile.status !== "active") { redirect to /pending }`.
