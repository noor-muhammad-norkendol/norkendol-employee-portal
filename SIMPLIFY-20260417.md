## What Changed

### 1. UserAutocomplete — Added 300ms debounce
**File:** `src/app/dashboard/onboarder-kpi/components/UserAutocomplete.tsx`
**What:** Added `debouncedSearch` state with 300ms delay before firing DB query
**Why:** Was firing a Supabase query on every single keystroke. Typing "Kenneth" = 7 queries. Now waits 300ms after you stop typing, then fires 1 query.
**If it breaks:** Autocomplete dropdown might not appear or might be delayed. Revert the `debouncedSearch` state and change all `debouncedSearch` references back to `search`.

### 2. KPIAdminTab — Fixed Supabase client creation
**File:** `src/app/dashboard/executive-intelligence/KPIAdminTab.tsx`
**What:** Changed `const supabase = createClient()` to `const [supabase] = useState(() => createClient())`
**Why:** Was creating a brand new Supabase client on every single render of the component. This caused the `useCallback`/`useEffect` to re-run every render (infinite loop risk).
**If it breaks:** KPI Admin templates might not load. Revert to `const supabase = createClient()`.

### 3. KPIAdminTab — Batch template saves
**File:** `src/app/dashboard/executive-intelligence/KPIAdminTab.tsx`
**What:** Replaced the `for` loop of individual insert/update calls with a single `.upsert()` call
**Why:** Was making up to 18 sequential DB round-trips (one per template). Now does 1.
**If it breaks:** "Save All Changes" might fail. The upsert relies on the unique index `(org_id, stage, contact_target, channel)`. If that index is missing, saves will fail. Check Supabase for the index `idx_onboarding_templates_unique`.

### 4. page.tsx — Parallel status mutations
**File:** `src/app/dashboard/onboarder-kpi/page.tsx`
**What:** Changed 3 sequential `await` calls in `moveStatus()` to `Promise.all([...])`
**Why:** The update, status log, and activity log are independent writes — no reason to wait for one before starting the next.
**If it breaks:** Moving a client to a new status might show errors. Revert `Promise.all([...])` back to 3 separate `await` lines.

### 5. ActionComposer — Removed dead function
**File:** `src/app/dashboard/onboarder-kpi/components/ActionComposer.tsx`
**What:** Removed `cycleRecipient()` function (was defined but never called)
**Why:** Dead code. The To/CC buttons use inline `setRecipients` calls instead.
**If it breaks:** It won't — function was never called.

### 6. WorkboardTable — Removed unused import
**File:** `src/app/dashboard/onboarder-kpi/components/WorkboardTable.tsx`
**What:** Removed `import { buildEmailMailto, buildTextMailto } from "./mailtoHelpers"`
**Why:** These functions are no longer used in WorkboardTable (email/text now goes through ActionComposer). The import was dead code.
**If it breaks:** It won't — import was unused.
